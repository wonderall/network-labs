# 3. 기억장치
### 메모리 구조 (Memory Layout)

# 1. Text 영역 (.text)

## 개요
프로그램의 실행 코드(기계어 명령어)가 저장되는 영역이다.

## 특징

- 프로그램 실행 파일에서 그대로 메모리에 로드됨
- 실행 중 내용 변경 불가(Read Only)
- CPU가 실행하는 명령어 존재
- 여러 프로세스가 공유 가능

## 저장되는 내용

- 함수 코드
- main()
- 라이브러리 코드
- 실행 명령어

### 예시

```c
int main() {
    printf("Hello");
}
```

`main()`과 `printf()`를 호출하는 코드가 Text 영역에 저장된다.

---

# 2. Data 영역

전역 변수와 static 변수가 저장되는 영역이다.

Data 영역은 다시 두 부분으로 나뉜다.

## Initialized Data (.data)

초기값이 있는 전역 변수

```c
int a = 10;
static int b = 20;
```

초기값이 있기 때문에 실행 파일에도 저장된다.

---

## BSS (.bss)

초기값이 없는 전역 변수

```c
int a;
static int b;
```

또는

```c
int a = 0;
```

실행 시 운영체제가 0으로 초기화한다.

---

## 특징

- 프로그램 시작부터 종료까지 유지
- 함수 종료와 관계없이 존재
- 프로그램이 종료되면 해제

---

# 3. Heap 영역

## 개요

동적으로 메모리를 할당하는 공간이다.

프로그래머가 직접 크기를 결정하여 사용한다.

## 사용 함수

### C

```c
malloc()
calloc()
realloc()
free()
```

### C++

```cpp
new
delete
```

### Java

```java
new
```

(Java는 Garbage Collector가 자동 해제)

---

## 예시

```c
int *p = malloc(sizeof(int));
```

Heap에 메모리가 생성된다.

---

## 특징

- 런타임 중 생성
- 크기를 자유롭게 변경 가능
- 직접 해제 필요(C/C++)
- 메모리 누수(Memory Leak) 발생 가능

---

# 4. Stack 영역

## Stack(스택)

### 개요

스택(Stack)은 **LIFO(Last In, First Out, 후입선출)** 구조의 메모리 영역이다.  
함수 호출 시 필요한 정보(지역 변수, 매개변수, 복귀 주소 등)를 저장하며, 함수가 종료되면 자동으로 제거된다.

---

## LIFO(후입선출)

가장 마지막에 저장된 데이터가 가장 먼저 꺼내지는 구조이다.

```text
Push(A)
Push(B)
Push(C)

┌─────┐
│  C  │ ← 가장 먼저 Pop
├─────┤
│  B  │
├─────┤
│  A  │
└─────┘
```

순서

```
Push A
Push B
Push C

↓

Pop → C
Pop → B
Pop → A
```

---

## Stack에 저장되는 정보

- 지역 변수(Local Variable)
- 함수 매개변수(Parameter)
- 레지스터(Register) 값
- **복귀 주소(Return Address)**

---

## 복귀 주소(Return Address)

함수가 호출되면 CPU는 **현재 실행 중인 다음 명령어의 주소**를 스택에 저장한다.

이를 **복귀 주소(Return Address)** 라고 한다.

함수 실행이 끝나면 저장해 두었던 주소를 이용하여 원래 실행하던 위치로 돌아간다.

---

### 함수 호출 예

```c
int main()
{
    job1();
}

void job1()
{
    job2();
}

void job2()
{
    job3();
}

void job3()
{
}
```

스택 변화

```text
job1 호출
Stack
┌────────────┐
│ main 복귀주소 │
└────────────┘

↓

job2 호출

┌────────────┐
│ job1 복귀주소 │
├────────────┤
│ main 복귀주소 │
└────────────┘

↓

job3 호출

┌────────────┐
│ job2 복귀주소 │
├────────────┤
│ job1 복귀주소 │
├────────────┤
│ main 복귀주소 │
└────────────┘
```

---

## 인터럽트 발생 시

프로그램 실행 중 인터럽트(Interrupt)가 발생하면 CPU는 현재 작업을 잠시 중단한다.

이때 현재 실행 위치를 다시 이어서 실행하기 위해 **현재 명령어의 복귀 주소(Program Counter, PC)와 CPU 상태(Register 등)를 스택에 저장**한다.

예를 들어,

```text
job1()
   ↓
job2()
   ↓
job3() 실행 중

↓

인터럽트 발생

↓

CPU
- Program Counter(PC)
- 상태 레지스터(PSW)
- 필요한 레지스터

↓

Stack에 저장

↓

인터럽트 서비스 루틴(ISR) 실행

↓

ISR 종료

↓

Stack에서 복귀 주소를 꺼내

↓

job3()의 중단된 지점부터 계속 실행
```

즉, 인터럽트가 발생해도 스택에 저장된 복귀 정보를 이용하여 원래 작업을 이어서 수행할 수 있다.

---

## Stack Overflow

스택의 크기를 초과하여 데이터를 저장하면 **Stack Overflow(스택 오버플로우)** 가 발생한다.

원인

- 무한 재귀 호출
- 너무 큰 지역 변수 선언
- 스택 크기 초과

예시

```c
void func()
{
    func();
}
```

함수가 계속 호출되어 스택이 가득 차면 Stack Overflow가 발생한다.

---

## 버퍼 오버플로우(Buffer Overflow) 공격

스택은 **버퍼 오버플로우(Buffer Overflow)** 공격과 매우 밀접한 관련이 있다.

예를 들어,

```c
char buf[8];
gets(buf);
```

입력

```
12345678901234567890
```

버퍼 크기보다 많은 데이터가 입력되면 인접한 메모리까지 덮어쓰게 된다.

```text
높은 주소
┌──────────────┐
│ Return Addr  │  ← 공격 대상
├──────────────┤
│ Saved EBP    │
├──────────────┤
│ buf[8]       │
└──────────────┘
낮은 주소
```

공격자가 버퍼를 넘쳐 쓰면 **복귀 주소(Return Address)** 를 악성 코드 주소로 변경할 수 있다.

함수가 종료되면 CPU는 변조된 복귀 주소로 이동하여 악성 코드를 실행하게 된다.

---

## 버퍼 오버플로우 방어 기법

- Stack Canary
- DEP(Data Execution Prevention)
- ASLR(Address Space Layout Randomization)
- NX Bit
- 안전한 함수 사용 (`fgets()`, `strncpy()` 등)

---

## 핵심 정리

| 항목 | 내용 |
|------|------|
| 자료구조 | LIFO(후입선출) |
| 저장 정보 | 지역 변수, 매개변수, 레지스터, 복귀 주소 |
| 생성 시점 | 함수 호출 |
| 제거 시점 | 함수 종료 |
| 인터럽트 | PC, 레지스터 등을 스택에 저장 후 ISR 실행 |
| 대표 오류 | Stack Overflow |
| 보안 | Buffer Overflow 공격의 주요 대상 |
| 성장 방향 | 높은 주소 → 낮은 주소(↓) |

## 저장되는 내용

- 지역 변수(Local Variable)
- 함수 매개변수(Parameter)
- 반환 주소(Return Address)
- 레지스터 정보

---

## 예시

```c
void func() {
    int x = 10;
}
```

`x`는 Stack에 저장된다.

함수가 종료되면 자동으로 제거된다.

---

## 특징

- 함수 호출 시 생성
- 함수 종료 시 자동 제거
- 매우 빠른 메모리
- 크기가 제한적
- Stack Overflow 발생 가능

---

# 메모리 성장 방향

```text
높은 주소
+----------------------+
|        Stack         |
|          ↓           |
|                      |
|                      |
|      Free Space      |
|                      |
|          ↑           |
|         Heap         |
+----------------------+
|        Data          |
+----------------------+
|        Text          |
+----------------------+
낮은 주소
```

- **Stack → 아래 방향(↓)으로 성장**
- **Heap → 위 방향(↑)으로 성장**

두 영역이 서로 만나면 메모리 부족(Out of Memory) 문제가 발생할 수 있다.


---

# 메모리 영역 비교

| 구분 | 저장 내용 | 생성 시점 | 제거 시점 | 읽기/쓰기 |
|------|----------|-----------|-----------|-----------|
| Text | 실행 코드 | 프로그램 시작 | 프로그램 종료 | 읽기 전용(Read Only) |
| Data | 전역 변수, static 변수 | 프로그램 시작 | 프로그램 종료 | 가능 |
| Heap | 동적 할당 메모리 | 실행 중 | 직접 해제 또는 GC | 가능 |
| Stack | 지역 변수, 함수 호출 정보 | 함수 호출 | 함수 종료 | 가능 |

---

# 변수 저장 위치 예제

```c
#include <stdio.h>
#include <stdlib.h>

int global = 10;        // Data
static int s = 20;      // Data

void func() {
    int local = 30;                     // Stack
    int *p = malloc(sizeof(int));       // p는 Stack
                                        // *p가 가리키는 메모리는 Heap
    *p = 100;

    free(p);
}

int main() {
    func();
    return 0;
}
```

| 변수 | 저장 위치 |
|------|----------|
| main() | Text |
| func() | Text |
| global | Data |
| s | Data |
| local | Stack |
| p(포인터 변수) | Stack |
| malloc()로 생성된 메모리 | Heap |

---

# 핵심 정리

| 영역 | 핵심 키워드 |
|------|-------------|
| Text | 실행 코드 |
| Data | 전역 변수, static 변수 |
| Heap | 동적 메모리, malloc/new |
| Stack | 지역 변수, 함수 호출 |

### 암기 포인트

- **Text** → 코드
- **Data** → 전역/static
- **Heap** → 동적 할당
- **Stack** → 지역 변수, 함수 호출
- **Stack은 아래로 성장(↓)**
- **Heap은 위로 성장(↑)**
