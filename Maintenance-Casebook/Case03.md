# 광 회선 이전 후 Uplink 연결 불가 - 광 포트 속도 인식 문제

## 작업 개요

건물 A는 기존에 건물 B를 경유하여 건물 C와 광 회선으로 연결되어 있었다.

건물 B 철거에 따라 건물 A와 건물 C를 직접 광으로 연결하도록 네트워크를 변경하는 작업을 수행하였다.

기존 스위치 설정은 변경하지 않았으며, 단순히 광 회선을 직접 연결하는 작업만 진행하였다.

그러나 작업 후 Uplink 광 포트가 활성화되지 않아 통신이 불가능한 상황이 발생하였다.

---

## 네트워크 구성

### 변경 전

```text
건물 A
   │
   │ Fiber
   │
건물 B
   │
   │ Fiber
   │
건물 C
```

### 변경 후

```text
건물 A
   │
   │ Fiber
   │
건물 C
```

---

## 증상

- 광 회선 직접 연결 후 Link Down
- Uplink 포트 비활성화
- 설정 변경 이력 없음
- 기존에는 정상 운영되던 구간

장애 복구를 위해 임시로 광 컨버터를 설치하여 Ethernet(1Gbps)으로 연결한 후 서비스를 우선 복구하였다.

---

## 원인 분석

설정 변경 없이 물리적인 연결만 변경하였기 때문에 설정(Config) 오류 가능성은 낮다고 판단하였다.

현장을 방문하여 스위치 설정을 확인한 결과,

- VLAN 설정 정상
- Trunk 설정 정상
- Interface 설정 정상
- 기존 Config와 동일

설정에는 이상이 없었다.

이후 인터페이스 상태를 확인한 결과, 광 포트가 10Gbps가 아닌 **1Gbps(SFP)** 로 인식되고 있는 것을 확인하였다.

자동 협상 상태에서는 링크가 올라오지 않았으며, 인터페이스 속도를 **1000Mbps**로 고정 설정한 후 정상적으로 Link가 활성화되었다.

---

## 확인 절차

1. 물리적 연결 상태 확인
2. 광 패치 상태 확인
3. Config 비교
4. VLAN 및 Trunk 설정 확인
5. Interface 상태 확인
6. SFP 인식 속도 확인
7. Interface Speed 변경
8. Link 정상 여부 확인

---

## 조치 내용

- 광 포트의 속도를 Auto에서 **1000Mbps**로 변경
- Link Up 확인
- 통신 테스트 수행
- 서비스 정상 복구

장애 발생 당시에는 서비스 연속성을 위해 광 컨버터를 이용한 1Gbps 임시 구성도 함께 적용하였다.

---

## 결과

- Uplink Link Up
- 건물 간 통신 정상
- 서비스 정상 운영

---

## 추가 검토 사항

설정에는 문제가 없었으며, 광 포트가 1Gbps로 인식된 원인에 대해서는 추가적인 확인이 필요하다고 판단하였다.

가능한 원인으로는 다음 사항을 검토 대상으로 남겼다.

- SFP 모듈 이상
- SFP 모듈 호환성 문제
- 광 회선 또는 광 패치 상태
- 상대 장비의 포트 설정
- 광 모듈 자체의 성능 저하

---

## 배운 점

- 설정 변경이 없는 경우에도 인터페이스 상태와 SFP 인식 정보를 반드시 확인해야 한다.
- Link Down 장애는 Config보다 물리 계층(Layer 1)에서 원인이 발생하는 경우가 많다.
- 서비스 중단을 최소화하기 위해 임시 우회 구성을 먼저 적용하고, 이후 원인을 분석하는 접근이 효과적이었다.

---

## 핵심 원인

**설정(Config)에는 문제가 없었으며, 광 포트가 예상과 달리 1Gbps로 인식되어 자동 링크가 성립되지 않았다. 인터페이스 속도를 1000Mbps로 설정하여 정상적으로 링크를 복구하였다.**

> **참고:** 본 사례에서는 속도 고정으로 서비스를 복구하였으나, 광 포트가 1Gbps로 인식된 근본 원인(SFP 모듈 이상, 호환성, 상대 장비 설정 등)은 추가 점검이 필요한 사항으로 기록하였다.
