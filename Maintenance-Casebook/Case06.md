# Case 06. VMware 서버 NIC 비활성화로 인한 인터페이스 Up/Down 반복

## 작업 개요

스위치 A와 VMware 서버가 이중화 환경으로 연결되어 운영 중이었다.

운영 구조상 해당 스위치 포트는 평상시 트래픽을 처리하지 않는 대기(Standby) 경로였으나, 특정 인터페이스에서 지속적으로 Link Up/Down이 반복되는 현상이 발생하였다.

장애 원인 확인을 위해 네트워크 장비와 서버를 순차적으로 점검하였다.

---

## 네트워크 구성

```text
        VMware Server
        +-----------+
        |   NIC1    |
        |   NIC2    |
        +-----------+
          │      │
          │      │
      Switch A  Switch B
```

- Switch A ↔ VMware (Standby 경로)
- Switch B ↔ VMware (Active 경로)

---

## 증상

- 특정 포트에서 Link Up/Down 반복 발생
- 로그에 Interface Up/Down 이벤트 지속 발생
- 운영 서비스에는 큰 영향 없음
- 물리 케이블 이상 징후는 확인되지 않음

---

## 원인 분석

우선 스위치 인터페이스 상태와 설정을 확인하였으나 특별한 이상은 발견되지 않았다.

이후 서버 담당자에게 이상 현상을 전달하고 VMware 서버 측 네트워크 설정을 확인하였다.

확인 결과 해당 NIC에서 다음과 같은 상태를 확인하였다.

- 네트워크 설정 정보 미입력
- NIC가 정상적으로 활성화(Up)되지 않은 상태

서버 측에서 NIC를 활성화(Up)하고 네트워크 정보를 다시 설정한 후 인터페이스의 Up/Down 반복 현상이 사라졌으며 링크가 정상적으로 유지되었다.

---

## 확인 절차

1. 스위치 Interface 상태 확인
2. Interface Log 확인
3. 물리 케이블 확인
4. 서버 담당자와 협업
5. VMware NIC 상태 확인
6. NIC 활성화 및 네트워크 설정
7. Link 안정성 확인

---

## 조치 내용

- VMware 서버 NIC 활성화
- NIC 네트워크 설정 입력
- 링크 상태 모니터링

---

## 결과

- Interface Up/Down 반복 현상 종료
- Link 상태 정상 유지
- 추가적인 이상 현상 미발생

---

## 배운 점

- Interface Up/Down 장애는 스위치뿐만 아니라 서버의 NIC 상태도 함께 확인해야 한다.
- 이중화 환경에서는 대기 링크라도 서버 측 설정 이상으로 지속적인 링크 이벤트가 발생할 수 있다.
- 네트워크 장비와 서버를 함께 점검하는 협업이 장애 해결에 중요하다.

---

## 확인되지 않은 사항

본 사례에서는 NIC를 활성화한 후 증상이 정상적으로 해결되었으나, **NIC가 비활성화된 정확한 원인은 확인하지 못하였다.**

장애 발생 당시 관련 로그가 충분히 남아 있지 않아 원인 분석에는 한계가 있었으며, 추후 동일 증상 발생 시 VMware 이벤트 로그와 운영체제 로그를 함께 확인할 필요가 있다.

---

## 핵심 원인

**VMware 서버의 NIC가 정상적으로 활성화되지 않은 상태였으며, NIC를 활성화하고 네트워크 정보를 재설정한 후 Interface Up/Down 반복 현상이 정상적으로 해소되었다. 다만 NIC가 비활성화된 근본 원인은 로그 부족으로 확인하지 못하였다.**
