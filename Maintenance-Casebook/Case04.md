# 서버실 스위치 직접 광 연결 후 통신 불가 - 서브넷 마스크(Prefix Length) 설정 오류

## 작업 개요

건물 A에서 서버실 스위치 A까지 광케이블을 직접 연결하여 네트워크를 구성하는 작업이 진행되었다.

건물 A에는 광을 지원하는 스위치가 설치되었으며, 해당 스위치와 서버실 스위치 A를 광케이블로 직접 연결하였다.

작업 완료 후 네트워크 통신이 되지 않는다는 요청을 받아 원인 분석을 진행하였다.

---

## 네트워크 구성

```text
건물 A
   │
   │ Fiber
   │
광 지원 스위치
   │
   │ Fiber
   │
서버실 Switch A
```

---

## 증상

- 광 링크 정상(Link Up)
- Ping 통신 불가
- 물리 연결 정상
- 광 포트 정상 동작

---

## 원인 분석

먼저 광 링크 상태를 확인한 결과 Link Up 상태였으며, 물리적인 연결에는 문제가 없었다.

이후 다음 항목을 순차적으로 점검하였다.

- IP 주소
- Default Gateway
- VLAN 설정
- 포트 상태

모든 설정이 정상으로 보였으나, 인터페이스의 Prefix Length를 확인하는 과정에서 서브넷 마스크가 네트워크 설계와 다르게 설정되어 있는 것을 확인하였다.

**기존 설정**

- Prefix Length : **/24**
- Subnet Mask : **255.255.255.0**

**설계 정보**

- Prefix Length : **/25**
- Subnet Mask : **255.255.255.128**

잘못된 Prefix Length로 인해 동일 네트워크에 있어야 하는 장비를 서로 다른 네트워크로 인식하여 정상적인 통신이 이루어지지 않았다.

---

## 확인 절차

1. 광 링크 상태 확인
2. Ping 테스트
3. IP 주소 확인
4. Default Gateway 확인
5. VLAN 설정 확인
6. Prefix Length(Subnet Mask) 확인
7. 서브넷 마스크 수정
8. 통신 테스트

---

## 조치 내용

인터페이스의 Prefix Length를 설계에 맞게 수정하였다.

- 기존 : **/24 (255.255.255.0)**
- 변경 : **/25 (255.255.255.128)**

설정 변경 후 Ping 테스트를 수행하여 정상 통신을 확인하였다.

---

## 결과

- Ping 정상
- Gateway 통신 정상
- 서비스 정상

---

## 배운 점

- IP 주소와 Default Gateway가 올바르더라도 서브넷 마스크가 잘못 설정되어 있으면 통신이 실패할 수 있다.
- 네트워크 장애 발생 시 물리 계층뿐만 아니라 IP 주소, Gateway, Prefix Length를 함께 확인하는 것이 중요하다.
- IP 주소만 확인하는 것이 아니라 **Prefix Length(Subnet Mask)**까지 반드시 검토해야 한다.

---

## 핵심 원인

**인터페이스의 Prefix Length가 설계와 다르게 /24로 설정되어 있었으며, 이를 /25(255.255.255.128)로 수정하여 정상적으로 통신이 복구되었다.**
