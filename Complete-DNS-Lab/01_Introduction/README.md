# 인터넷은 어떻게 목적지를 찾을까?

웹 브라우저를 실행한 후 주소창에 다음과 같이 입력합니다.

```

www.google.com

```

또는

```

www.naver.com

```

그리고 Enter를 누르면 몇 초도 지나지 않아 웹페이지가 화면에 나타납니다.

우리는 너무 익숙해서 의식하지 못하지만, 실제로는 여러 단계의 네트워크 통신이 순서대로 진행됩니다.

가장 먼저 수행되는 작업은 DNS입니다.

웹 브라우저는 **도메인 이름만으로는 통신할 수 없기 때문입니다.**

---

# 사람과 컴퓨터는 서로 다른 언어를 사용한다.

사람은 이름을 기억합니다.

```

김철수

```

하지만 휴대전화는

```

010-1234-5678

```

을 이용하여 통신합니다.

인터넷도 완전히 같습니다.

우리는

```

www.naver.com

```

을 기억합니다.

그러나 컴퓨터는

```

223.130.xxx.xxx

```

처럼 숫자로 구성된 IP 주소만 이해합니다.

즉,

사람은 **도메인(Domain Name)** 을 사용하고,

컴퓨터는 **IP 주소(IP Address)** 를 사용합니다.

---

# 그림으로 이해하기

```
             사람

      www.naver.com

             │

             ▼

      DNS Server

             │

             ▼

      223.130.xxx.xxx

             │

             ▼

        Router

             │

             ▼

       Web Server
```

DNS는 사람이 사용하는 이름을 컴퓨터가 사용할 수 있는 IP 주소로 변환하는 시스템입니다.

이 과정을 **이름 해석(Name Resolution)** 이라고 합니다.

---

# DNS란 무엇인가?

DNS(Domain Name System)는 인터넷에서 사용하는 분산형 데이터베이스 시스템입니다.

주요 역할은 다음과 같습니다.

- 도메인 이름을 IP 주소로 변환
- IP 주소를 도메인 이름으로 변환(Reverse DNS)
- 메일 서버 조회
- 서비스 위치 조회
- 클라우드 서비스 연결
- 로드 밸런싱 지원

많은 사람이 DNS를 단순히 "도메인을 IP 주소로 변환하는 서비스" 정도로 이해하지만,

실제로 DNS는 인터넷 전체를 연결하는 핵심 인프라입니다.

웹 서비스뿐 아니라 이메일, Active Directory, Kubernetes, CDN, 클라우드 환경까지 대부분의 시스템이 DNS를 기반으로 동작합니다.

---

# 왜 DNS가 필요한가?

DNS가 없다면 사용자는 모든 서버의 IP 주소를 직접 기억해야 합니다.

예를 들어

```
203.248.xxx.xxx
```

또는

```
142.xxx.xxx.xxx
```

와 같은 IP 주소를 외워야 합니다.

현실적으로 불가능합니다.

하지만 DNS가 중요한 이유는 단순히 기억하기 쉽기 때문만은 아닙니다.

---

## 서버의 IP 주소는 변경될 수 있다.

기업에서는 다음과 같은 이유로 서버의 IP 주소가 변경됩니다.

- 서버 교체
- 장비 업그레이드
- 데이터센터 이전
- 클라우드 이전
- Load Balancer 추가
- 서비스 확장
- DR(Disaster Recovery)

만약 사용자가 IP 주소를 직접 입력했다면

IP 주소가 변경될 때마다 모든 사용자에게 새로운 주소를 안내해야 합니다.

하지만 DNS를 사용하면

관리자는 DNS 레코드만 변경하면 됩니다.

사용자는 항상 동일한 도메인을 사용할 수 있습니다.

---

## 실무 사례

기존 서버

```
portal.company.com

↓

10.10.10.100
```

신규 서버

```
portal.company.com

↓

10.20.20.100
```

사용자는

```
https://portal.company.com
```

만 입력합니다.

DNS의 A 레코드만 수정하면 사용자는 자동으로 새로운 서버로 연결됩니다.

이것이 DNS가 인터넷에서 반드시 필요한 이유입니다.

---

# DNS는 언제 동작할까?

많은 사람이

웹 브라우저가 HTTP 요청을 먼저 보낸다고 생각합니다.

실제로는 그렇지 않습니다.

순서는 다음과 같습니다.

```
사용자

↓

도메인 입력

↓

DNS Query

↓

IP 주소 획득

↓

TCP 3-Way Handshake

↓

HTTP Request

↓

HTTP Response
```

즉,

DNS가 완료되어야 TCP 연결을 생성할 수 있습니다.

TCP 연결이 만들어져야 HTTP 또는 HTTPS 요청이 가능합니다.

DNS는 인터넷 통신의 첫 번째 단계입니다.

---

# OSI 모델에서 DNS

| 계층 | 프로토콜 |
|------|-----------|
| Application | DNS HTTP HTTPS SMTP FTP |
| Transport | TCP UDP |
| Network | IPv4 IPv6 |
| Data Link | Ethernet |
| Physical | UTP Fiber Wireless |

DNS는 Application Layer에서 동작합니다.

일반적인 DNS 질의(Query)는 UDP 53번 포트를 사용합니다.

다만 다음과 같은 경우에는 TCP 53번 포트를 사용합니다.

- Zone Transfer
- DNSSEC
- 응답 크기가 UDP 제한을 초과하는 경우

---

# 이번 장 핵심 정리

- 인터넷은 도메인이 아니라 IP 주소를 이용하여 통신한다.
- 사람은 도메인을 기억하지만 컴퓨터는 IP 주소를 사용한다.
- DNS는 도메인을 IP 주소로 변환하는 시스템이다.
- DNS는 Application Layer에서 동작한다.
- DNS는 TCP 연결보다 먼저 수행된다.
- 대부분의 인터넷 서비스는 DNS를 기반으로 동작한다.
